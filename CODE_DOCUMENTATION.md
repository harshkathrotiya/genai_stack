## üîÑ Data Flow Architecture

### Workflow Creation Flow
```
1. User drags component ‚Üí ComponentLibrary
2. Component added ‚Üí WorkflowBuilder state update
3. User connects components ‚Üí React Flow edge creation
4. Configuration changes ‚Üí ConfigurationPanel updates
5. Build Stack clicked ‚Üí API call to validate and save
6. Workflow stored ‚Üí PostgreSQL database
```

### Chat Execution Flow
```
1. User sends message ‚Üí ChatModal
2. API call to workflow execution ‚Üí WorkflowService
3. Component execution order determined ‚Üí Topological sort
4. Sequential component processing:
   ‚îú‚îÄ User Query: Pass through message
   ‚îú‚îÄ Knowledge Base: Vector search in ChromaDB
   ‚îú‚îÄ LLM Engine: Generate response with context
   ‚îî‚îÄ Output: Format and return response
5. Response displayed ‚Üí ChatModal
```

### Document Processing Flow
```
1. File upload ‚Üí ConfigurationPanel
2. File validation ‚Üí DocumentService
3. Text extraction ‚Üí PyMuPDF/python-docx
4. Text chunking ‚Üí Overlapping chunks strategy
5. Embedding generation ‚Üí OpenAI/Gemini embeddings
6. Vector storage ‚Üí ChromaDB collections
7. Database update ‚Üí Document processed status
```

**Solution**: Dynamic form generation based on component type

```javascript
const renderConfigField = (key, value, type = 'text') => {
    switch (type) {
```

### 3. Workflow Validation
**Challenge**: Ensuring workflow components are properly connected
**Solution**: Graph validation algorithm

```javascript
const validateWorkflowConnections = (nodes, edges) => {
    const nodeTypes = nodes.map(n => n.data.componentType);
    const hasRequiredComponents = 
        nodeTypes.includes('user-query') && 
        nodeTypes.includes('output');
    
    const hasConnections = nodes.length <= 1 || edges.length > 0;
    return hasRequiredComponents && hasConnections;
};
```
